## Recommended reading order

1. Start with Architecture Decisions to understand package boundaries and tool choices.
2. Review Design Tokens, Responsive Behavior, Motion, and Dark Mode to align new UI work with system primitives.
3. Read Accessibility and Testing before adding or changing component behavior.
4. Use Component Usage Guidelines and the live component stories during implementation.
