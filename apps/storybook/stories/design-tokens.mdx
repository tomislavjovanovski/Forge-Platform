<CardGrid>
  <Card title="Semantic first">
    Use semantic families such as <code>brand</code>, <code>success</code>, and <code>critical</code> to preserve intent across themes and future redesigns.
  </Card>
  <Card title="Interactive layer">
    Interactive text, background, border, and state tokens capture the actual UI contract used by components instead of leaking raw palette choices everywhere.
  </Card>
  <Card title="Theme-ready">
    Dark mode values are represented as explicit overrides, which keeps theme switching predictable and avoids ad hoc component-level inversions.
  </Card>
</CardGrid>
