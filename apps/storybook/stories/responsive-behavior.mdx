## Authoring rules

<CardGrid>
  <Card title="Start narrow">
    Begin with the smallest useful layout and add complexity as space increases. This keeps information architecture honest and avoids desktop-first overflow fixes.
  </Card>
  <Card title="Protect hierarchy">
    Priority actions, status, and core content should remain visible on smaller screens. Secondary utilities can collapse or move, but intent cannot disappear.
  </Card>
  <Card title="Scale density deliberately">
    More columns do not automatically mean better use of width. Use larger breakpoints to improve scanability, comparison, and workflow continuity.
  </Card>
</CardGrid>
