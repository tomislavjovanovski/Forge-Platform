## Strategy summary

<CardGrid>
  <Card title="Semantic substitution">
    Components should read from semantic roles like text, background, border, and state tokens. Theme changes should swap token values rather than rewrite component structure.
  </Card>
  <Card title="Theme parity">
    The same information hierarchy should survive both themes. Dark mode is not an alternate product design; it is a color-system translation of the same interface contract.
  </Card>
  <Card title="Contrast discipline">
    Secondary text, dividers, and subtle states often fail first in dark themes. Validate contrast there instead of checking only primary text on primary backgrounds.
  </Card>
</CardGrid>
